Software de Versionamento

Sistema que tem por objetivo realizar o controle de versões de um código ou documento.

É possível realizar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.

Tipos de software de controle de versão:

Centralizado:
Possuí um repositório central e um servidor com controle de versões
Depende do servidor
Software subversion
Distribuído:
Possuí um repositório local com controle de versões
Atualizações no repositório central
Independene do servidor

Comandos Git

  • commit: Torna permanente um conjunto de alterações em um repositório
  • push: Envia alterações para p servidor repositório remoto
  • fetch: Verifica se seu repositório local está sincrona com o remoto
  • pull: Sincroniza o repositório local com o remoto atualizando as versões
